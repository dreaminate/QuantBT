# 机构级 Agent OS · 成品环节前沿研究（纵向深挖）

> 简介：本目录是对上级 [../README.md](../README.md) 那套「40-agent 横向蓝图」的逐环节**纵向深挖**——横向蓝图回答「整条治理脊柱由哪些器官组成」，本目录回答「每个成品环节，世界上最严谨的做法到底长什么样」。36 个环节，每个都走过同一条 workflow：**研究检索（前沿 SOTA + 关键论文带 URL + 机构最佳实践）→ 对抗式事实核查（逐条降权、点名撤稿/夸大/二手/不可外推）→ 综合落盘**，全程 Opus 4.8。产物刻意限定为**纯前沿研究 + 概念级架构推荐 + 少量架构级伪代码**，**不含 `file:line` 接线与实施计划**——那是下一轮的事。每份第 7 节保留了对抗核查的降权词（夸大/二手/撤稿/不可外推/厂商自利等），**任何引用都必须按降权后的口径理解**，不得把被降权的数字当既定事实。

## 怎么读

先读本页拿全局——尤其是「跨环节贯穿主题」和「对抗核查反复抓到的降权类型」两节，它们是 36 份共识的提炼；再按需点开单份 dossier。**每份的第 7 节（降权/争议/陷阱）和第 8 节（开放问题）是重点**：第 1 节给你定位，但真正的判断力藏在第 7、8 节——它们告诉你哪些「SOTA」其实是单篇未复现预印本、哪些数字是厂商营销、哪些方法只在合成环境里赢过。把第 7 节当成「引用前的免责声明」来读。

## 跨环节贯穿主题

把 36 份的判断压缩成几条反复出现的共识：

1. **因子轨架构已有定论（24/25/26/28 收敛）**：DL/ML 模型**本体进模型注册表**（存 `.pkl`/`.pt`+护照+训练快照），其**输出登记为信号进因子库**——公式因子与 ML/DL 输出在因子库里同居一处、地位平等；**两库由一份窄信号契约（instrument×time×score + shift + PIT + 输出指纹 + 模型版本反向引用）解耦并保持纯净**，因子库内绝不做组合；**所有排列组合/集成上移到信号层与策略层**去做。把 `.pt` 当一个因子塞进因子库是范畴错误，会逼因子库背上版本/血缘/重训的重量。

2. **「确定性≠可复现 / 谱系≠信任 / 审批门≠不橡皮图章 / 统计闸只与诚实的 N 一样可信」——一组反复出现的诚实边界**：01 划清确定性 DAG 给的是持久执行不是可复现；03 划清谱系只回答「怎么来的」不回答「可不可信」；04 划清门存在不代表不会被橡皮图章（决定因素是审批人激励）；14/15/23/28 一致指出 DSR/PBO/多重检验全部押在「有效独立试验数 N」上，而 N 在 agent 自动搜索下近乎不可数——统计闸再漂亮，也只与喂给它的那个诚实的 N 一样可信。

3. **安全 / 数据可信 / 留出集隔离必须前移到脊柱（06/09/05）**：安全不能当 P5 事后过滤（提示注入不可根治、实盘密钥绝不进上下文）；数据可信地基（双时态 PIT、反 look-ahead/survivorship、研究-实盘同源）必须在第一天就立起来；可证伪假设的预注册与「最终 OOS 只碰一次」的冻结必须落到数据访问层，否则「冻结」只是名义。

4. **单用户（D4）语境下，别堆企业级治理机器、要最小可行子集**：22/26/31/33 反复点破同一个张力——SR 11-7/NIST/EU AI Act 这套机构脊柱默认一个「有职责分离的组织」，而本项目「既是作者又是审批人」的单 operator 让「独立验证」「谁批了谁的审计轨」部分变成表演；治理外壳（每因子自动 SHAP/护照/不可变 lineage）的算力与维护负担可能吞掉它治理的标的本身的边际价值。原则是按风险比例取最小可行子集，而非把治理栈做得比交易系统还重。

5. **美国已发表研究不可直接外推 A股微结构**：几乎每一份的第 8 节都在喊同一句——T+1、10%/20% 涨跌停、频繁停牌、ST 制度、强散户结构、做空近乎不可行，会系统性破坏所引方法的核心假设（高斯发射、平方根冲击律的「连续可成交」、PSR/DSR 的 IID 平稳、空头腿可交易性、PIT 披露口径）。所有 SOTA 数字都来自美/德/日股或加密，A股迁移证据**普遍缺位**，必须显式标注适用边界而非默认通用。

6. **「统计上真 ≠ 经济上能赚」是贯穿因子轨与方法学的红线**：14/15/16/17/24/27/30 一致强调，过了 DSR/PBO/CSCV 闸门不等于覆盖了 look-ahead、survivorship、regime 漂移与交易成本——成本/容量/冲击审计必须与统计体检**串联**，否则会放行一批「统计真、经济假」的策略。

7. **监管叙事须以 2026 现实为准，且对本项目多为「借鉴精神」而非「强制约束」**：SR 11-7 已于 2026-04-17 被 SR 26-2 取代，且 SR 26-2 **明确把生成式/agentic AI 排除在适用范围外**；EU AI Act 才是真正具约束力的硬法但被多份忽略；而这些框架的适用对象是受监管金融机构，对单用户个人研究系统多无法律强制力。用机构光环给方案镀金是反复被点名的风险。

## 对抗核查反复抓到的降权类型

36 轮对抗核查里，最高频被抓的几类「不诚实引用」：

1. **撤稿/preprint 当既定事实**：最典型是 **arXiv 2512.11913（拥挤动态缩量）已被作者撤稿**（14/15 明确登记，所幸两份均未引用踩雷）；以及大量「2026 年 2–3 月刚挂出的单/双篇未评审预印本」被当成「金融落地证据正在出现」（17 的三篇金融 conformal 落地证据、19 的非齐次 HMM「加密前沿」均属此类）。

2. **厂商营销页/利益相关来源当中立权威**：01 的 ThruWire/Resonate/Diagrid/Temporal/DBOS「厂商来源高度集中却被当独立证据收敛」；11/16 的 mlfinlab 被点破「已闭源、All-Rights-Reserved、禁商用」却被当可商用开源参考；15 把厂商页（modelop.com）当 SR 11-7 一手监管文件。

3. **跨域/跨辖区监管模板硬搬**：**EU AI Act 第 12 条被过度声称**（把生物识别专属的细粒度日志义务泛化到所有高风险系统、把分析师「SHOULD」升格为法条「MUST」要求 tamper-evident）；SR 11-7/SR 26-2 反复被外推到「单用户零售产品也适用」「强制 model card」，而 SR 26-2 恰恰把 agentic AI 划出范围、转向风险比例化减负。

4. **精度过陈述 / 二手数字硬编**：**DSR 是「标度/选择偏差修正」被误说成「修复系统性低估」**（多份反复纠正，误差符号取决于真实横截面方差）；**平方根冲击律的 δ≈0.5 有 AQR 争议**（18 同时指出新证据 Sato-Kanazawa 2024 反而支持 δ=0.5，研究把工程建议与学术争议混为一谈）；pod 级 kill/scale 阈值 5%/7.5% 是二手数字应做软参数；AQR 一串两位小数 bps 无法独立溯源属「未核实的伪精确」。

5. **方向被讲反 / 数字被造**：02 的 DFAH「r=−0.11」与一手论文的「r=+0.45」**符号相反**（最严重的造数）；08 的 Ambig-SWE 被引为「多问无效」而原文 headline 是「提问提升最多 74%」；13 的 Vasconcelos「难任务下解释增加过度依赖」与原文「难任务下解释减少过度依赖」**完全倒置**；19 的 JM/HMM Sharpe 数字整组与论文 Table 4 对不上且内部不自洽。

> 共性：**CPCV 仅在合成受控环境（Heston 等）显著优于 walk-forward**，被多份反复降权为「更强默认而非铁律」；**DSR 是标度修正非低估修复**；以及「负面存在断言」（「据我检索没有任何系统……」）无法被证实只能被证伪——这些是引用本目录任何「SOTA」时最该警惕的口径陷阱。
