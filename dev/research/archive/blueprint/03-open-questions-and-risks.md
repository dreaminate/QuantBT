# 开放问题与风险（蓝图自身的盲点）

> 这是完备性复核 agent 对**总蓝图本身**的对抗式审查。它先独立把 6 个"洞"对源码二次确认（全部属实），然后专找蓝图**停止观察的地方**。这些不是"以后再说"——是**会变成下一批"已验证的洞"的地方**，必须在动工前纳入。

## 8 个盲点

### (1) "流程即信任"这个承重前提，自己没被预注册
蓝图把一切押在"process IS the trust"上，却从未提出**验证这个论断**的办法——它甚至自己把"谱系→恰当信任"标为待验假设，然后在其上建了整整 5 个阶段。**这个信任论断本身是未预注册的，违反了它自己的纪律。**
→ **行动**：在建脊柱前，给"流程即信任"一个可证伪指标 / 用户研究代理 / 反向校准检查。

### (2) 确定性 vs LLM 非确定性——是断言，不是工程
蓝图承诺"确定性 DAG / 可时间旅行 / 可回放"，但 clam、researcher、verifier 等 **LLM 节点固有非确定性**。没有 prompt/版本/seed/temperature 固定、没有响应缓存、没说明"混入确定性阈值去 LLM 自偏好"如何与随机 LLM 共存。**GOAL §1.2 的可复现保证（同代码+dataset+seed → ±1e-6）在 LLM 进入图节点的那一刻被静默破坏，蓝图从未调和这一点。**

### (3) Agent OS 的成本/延迟/失败经济学完全缺席
持久 DAG + 挂起数天的审批节点 + 因子海/文献海的 orchestrator-worker fan-out + 独立 challenger 在不同种子/切片上重跑 CPCV = **一笔巨大且反复的 LLM+算力账单 + 长运行进程可靠性问题**。无 token 预算、无成本上限、除"checkpoint"外无节点失败/部分恢复语义、无"审批中途 LLM API 宕机怎么办"的故事。

### (4) 实盘资本下的对抗安全模型严重欠覆盖
系统将持有 **Binance 交易 key**，但红队只覆盖过拟合（注入未来数据 / "换种子重跑"）。**完全没有**：经摄入文献/新闻的 **prompt-injection**、**tool-abuse**（能接线执行的 agent 可下单）、因子海**数据投毒**、探索 agent 与锁定 OOS 权限层的**特权混淆**。P5 才提 MCP/HMAC/withdraw-deny"控攻击面"——**给定实盘资本，这是倒置的**。

### (5) 人群谱不对称
L1–L4 翻译器与苏格拉底澄清官对**小白**和**经济学者**开发得很好，但 **quant/高级用户被欠服务**：无绕过漏斗的合法**探索研究逃生舱**、无 notebook/代码路径、无"我清楚自己在做什么，直接给我原始产物"模式。一条不可绕过、强制每个想法过预注册+HITL 的脊柱，对真实 quant 是**官僚摩擦**；蓝图自己的 confirmatory/exploratory 标记**从未接到更轻量的探索通道**。还缺"经济学者"与"quant"之间的**中间人群**（读统计但不读代码的 PM/风控经理），以及多用户/角色分离（creator/approver/allocator 是器官，但使 `approver≠creator` 可强制的人类 RBAC 是假设而非规格）。

### (6) 若干路线图依赖是高风险，却被当成低风险接线回填
- **honest-N 的 N_eff**（相关聚类估有效独立试验数）被定位为"最低工程量、最高杠杆"，**实为有争议的硬统计问题**——估错任一方向都会**静默重新击穿**它喂养的每一个多重检验校正。这是伪装成接线的深方法学任务。
- **锁定 OOS 的"权限层技术上无法访问"硬隔离**依赖一个**单机 DuckDB/Parquet 本地优先应用里并不存在的真实访问控制边界**——按"技术"而非"约定"强制，可能需要产品明确回避的架构（用户能读 Parquet = 能读 OOS）。
- **P3 live 监控**依赖真实 live/paper run 产出日 PnL——A股仅 paper、crypto 依赖 Binance 实盘链路先可信。P3 门控于 P0-P2 **且**门控于真实资本部署，线性 P0→P5 排序掩盖了这条交叉依赖。

### (7) 整片缺席的维度
- **(a) 数据可信度/source-freshness 没有作为漏斗的一等闸门节点** → garbage-in 可通过所有下游统计闸门。
- **(b)** 喂给 agent 的因子海/文献海本身的 PIT/幸存者偏差正确性，**未被绑定**到已做的 universe PIT 工作。
- **(c) A股程序化交易合规节点**（CSRC 300笔/秒、20000笔/日）缺席治理漏斗，尽管是硬性外部约束。
- **(d) 五道 HITL 闸门的决策疲劳**：面对五个长达数天的 interrupt-approve 闸门，真实用户会**橡皮图章**，反噬"人供 go/no-go 判断"；缺低风险审批的批处理/委派设计。
- **(e) 冷启动/空历史**：honest-N、谱系信任、champion/challenger 都假设已有 run 语料；新用户**第一个策略 N=1 且无挑战者历史**，漏斗第一天的行为未定义。
- **(f)** 组合/执行链路里的真实撮合现实未被点名：成交假设、部分成交、停牌/熔断、**A股 T+1 与涨跌停**对 `shift(1)` 与组合的影响。

### (8) 用词纪律不均匀
跨组综合已经对多条论断降权（"谱系→信任""DSR 标度修正非修复""FDR 不解决发表偏误""平方根律有争议""2512.11913 撤稿→仅监控""pod 5%/7.5% 二手"——这种自我怀疑是优点）。但应用不均：**"确定性 DAG 天然可回放"被当事实**（尽管有上述 LLM 非确定性问题）；"orchestrator-worker + subagent 写文件避免传话"直接从 Cognition/MAST 先验断言、无项目级证据；"流程即信任"作为承重前提却是全文证据最弱的。

## 弱论断清单（引用时必须保留限定）

- "流程即信任"对读不懂代码用户是承重前提，却是全文**证据最弱**的论断——无实验、无用户研究代理、无可证伪指标。
- "确定性 DAG 天然可时间旅行/可回放/可审计"与 **LLM-in-node 非确定性自相矛盾**。
- "orchestrator-worker 并行 + subagent 写文件避免传话"无项目级证据。
- honest-N 的 N_eff 是**有争议的硬统计问题**，伪装成接线。
- 锁定 OOS"技术上不可访问"依赖本地优先架构里**不存在的权限边界**。
- 平方根冲击律 Y~0.5/crypto~0.9 为二手且 **AQR 有争议**；敏感性区间外的稳健性未交代。
- DSR `var_sr_hat` 的估计本身在小样本/相关试验下不稳；退化近似与精修在典型 N 下的数值差异是否实质影响判定，未说明。

## 高风险依赖（线性排序掩盖的交叉依赖）

| 依赖 | 风险 |
|---|---|
| honest-N 自动累加 + N_eff 相关聚类 | 被定位为低工程量，实为深统计依赖；估错即静默归零所有多重检验校正——整条防过拟合链的**单点信任根基** |
| 锁定 OOS 硬隔离 | 依赖单机本地优先架构下**不存在的真实权限边界**；与 §1.1"本地优先/开放落盘/用户可独立审计导出"**结构性张力** |
| P3 live 监控闭环 | 依赖真实 live/paper run 持续产日 PnL；门控于 P0-P2 **且** 门控于真实资本部署 |
| 确定性 DAG 骨架 (P1) | 依赖先解决 LLM 节点非确定性，否则"可回放/时间旅行/审计"落空，连带谱系总线与 Verifier 失效 |
| Verifier 异模型独立重跑 (P1) | 依赖第二个 LLM 供应商/成本预算 + "混入确定性阈值"这一未工程化机制；供应商单点或成本失控 → 独立验证退化为同模型自评 |
| MCP/HMAC/withdraw-deny 安全模型 | 被放 P5 可选，但实盘 key + 能下单的 agent 自 P0 在场——**依赖顺序倒置**，攻击面在脊柱建成时即暴露 |
| InstrumentSpec/日历/bitemporal | 被排到 P3/P4，但成本模型(P2)、回测撮合、universe 解析都隐式依赖正确的合约规格与日历——**方法学纵深建在尚未一等化的资产元数据之上** |
| 整个脊柱依赖 LLM 持续可用与稳定 | reAct/clarifier/translator/verifier 节点无 LLM 降级/离线/限流时的故障模式设计——**单机本地优先与强 LLM 依赖之间无 fallback 路径** |
