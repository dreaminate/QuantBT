02 · 边界与不可违反约束

下列为已确认产品决策，前端实现须把它们编码为校验规则 / 不可变 UI 约束，而非提示文案。

B1市场组合 与 投资标的池 是两个独立节点，经 Tradable Domain 求交：TradableDomain = InvestmentUniverse ∩ MarketConstraints。支持显式 Union/Intersection/Difference 节点。
B2投资标的池只定义允许交易哪些标的，不含最终权重。
B3Model 只从 Model Registry 引用（model_id@version + Schema + Artifact 策略）；策略台不实现训练画布，重训去 Model 台。
B4因子可直接产生信号，也可进入 Model，或与 Model 输出组合。
B5Model 可直接输出 Target Portfolio，但 UI 仍强制其后经过：Position Validation → Portfolio Risk → Final Risk Gate → Execution。
B6Final Risk Gate 不可删除、不可绕过。任何到 Execution 的路径必须穿过它；画布层禁止建立绕过边。
B7同一张图表达 Backtest / Paper / Live；已发布 Live 版本只读，编辑触发 Fork 新草稿。
B8Ask 先展示 Ghost Flow + Diff，提交后应用；Auto / Bypass 直接改草稿但必须事务化、整轮可 Undo、有 Patch ID + 快照。Bypass ≠ 绕过 Final Risk Gate / 发布审批。
B9不伪造真实后端执行；所有 Mock 数据与模拟行为必须显式标注（UI 角标 MOCK）。

端口类型系统（每端口可查看 9 维）：port_name · data_type · schema · frequency · asset_scope · time_scope · currency · nullable · required。连线兼容性 5 态：

● 完全兼容 ◐ 可自动适配 ⚠ 需转换节点 ✕ 不兼容 ? 待后端校验

典型校验：日频输出不可隐式接分钟级输入 · 截面因子需同期多标的 · 股票专用因子不可直接用于黄金 · 非 PIT 数据不可用于历史回测 · Model 输入 Schema 必须匹配 · TargetPortfolio 不应误接 SignalIntent 口 · 不允许绕过 Final Risk Gate 进入执行（B6）。校验信息常驻 Inspector + 底部工作台，可定位，不只在 Toast。

07 · 节点状态机

idle ─edit→ dirty ─validate→ validating ─┬→ valid ──run→ queued → running ─┬→ succeeded
                                          └→ invalid                        ├→ warning
 succeeded/valid ─(上游改动)→ stale ─revalidate→ validating                 ├→ failed
 queued/running ─cancel→ cancelled                                          └ (retry → queued)
idle 灰 dirty 琥珀 validating/queued 蓝 valid/succeeded 绿 running 脉冲橙 warning invalid/failed 红 stale 斜纹 cancelled

运行入口：Validate · Run Node · Run From Here · Run Selection · Run Full Graph · Dry Run · Cancel · Retry · Compare Runs。状态以非颜色冗余表达（图标+文字），满足可访问性。

08 · 画布状态模型（分桶，勿合并为单一 store）

// 1 服务端状态（React Query）  graphQuery · runQuery · registryQuery · modelRegistryQuery
// 2 画布持久状态（持久化 / 进 Undo 栈）
graph: { nodes: Record<id,NodeInstance>, edges: Record<id,Edge>, stages: Stage[], subgraphs }
// 3 视口/UI 临时态（不进 Undo）  viewport:{panX,panY,zoom} · marquee · hoverPort · linking · dockTab · panelSizes
// 4 选中态            selection:{ nodeIds[], edgeIds[], subgraphPath[] }
// 5 Agent 提议态     proposal:{ patchId, ghosts:GhostOp[], diff, affected[], status } | null
// 6 Patch / Undo 态  undoStack:Patch[] · redoStack:Patch[]  (整轮 Agent 修改 = 一个 Patch)
// 7 Runtime 态       runtime:'backtest'|'paper'|'live' · readonly:boolean(live→true)
// 8 回测 Trace 态    activeRunId · lineageFocus:{ nodeId?, tradeId?, t? } · contributionByNode

性能：大量节点虚拟化（仅渲染视口内）· selector 局部订阅 · 边/节点增量更新 · 大子图折叠为摘要 · 大型 Trace 渐进加载。Undo/Redo 以不可变 Patch 表达（apply/invert）。

09 · Agent Proposal / Patch 状态机

ASK   : idle → proposing → preview(Ghost+Diff 可见) ─┬ acceptAll → applying → applied(入Undo栈,1 Patch)
                                                          ├ acceptPartial(选 GhostOp 子集) → applying → applied
                                                          ├ reject → idle
                                                          └ refine(继续对话改建议) → proposing
AUTO  : idle → applying(草稿直接改) → applied  · 展示本轮变更摘要 + Patch ID + 受影响节点 + 快照
BYPASS: 同 AUTO，可跨节点批量改，不逐项询问
全模式: 整轮可 Undo（撤销整个 Patch）· 可恢复到会话开始前快照
红线 : Auto/Bypass 不可删除/绕过 FinalRiskGate(B6)，不可越过发布审批(B7/B8)

GhostOp 种类：addNode · addEdge · removeNode(删除标记) · setParam(参数 Diff) · setFormula(公式 Diff) · setCode(Code Diff) · relayout。Preview 同时呈现影响范围与风险提示。
